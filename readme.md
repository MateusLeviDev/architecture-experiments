#### PubSub - Use Cases
real time stream data ingestion from application and IoT devices
parallel processing and workflows, multiple tasks can be executed through Cloud functions.
Enterprise eventbus for sharing events across organizations.
replicate data among databases

- topic: a named resource to which messages are sent by publishers
- subscription: a named resource representing the stream of messages from a single, specific topic
- message: combination of data and attributes
- message attributes: in key, values pair about message
- publisher: an application that create messages for topics
- subscriber: an application that receives messages from topics through subscriptions
- acknowledgement: a signal received from subscriber to pubsub reg message delivery
- push and pull: message delivery methods by pubsub

PubSub allows services to communicate asynchronously, enables us to create systems of event producers and consumers, called publishers and subscribers
